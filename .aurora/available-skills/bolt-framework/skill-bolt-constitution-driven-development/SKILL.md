# Constitution-Driven Development

## When to Use

- Initializing new Bolt Framework projects
- Reading project constraints before code generation
- Resolving conflicts between agent suggestions

## What is the Constitution?

**Location:** `.aurora/memory/constitution.md`

The Constitution is the **project's DNA** — a single source of truth defining:

- Technology stack (languages, frameworks, libraries)
- Architecture patterns (layering, boundaries, dependencies)
- Quality standards (coverage thresholds, mutation scores)
- Development practices (TDD, BDD, code review rules)
- Infrastructure policies (environments, security, observability)

**It is LAW for all AI agents** — no agent can violate constitution rules.

## Directive Hierarchy

```
┌─────────────────────────────┐
│     1. CONSTITUTION         │  ← Project-specific law
│  (.aurora/memory/          │
│   constitution.md)          │
├─────────────────────────────┤
│     2. SKILLS               │  ← Domain-specific practices
│  (.github/skills/*)        │
├─────────────────────────────┤
│     3. AGENTS               │  ← Specialized assistants
│  (.github/agents/*)        │
└─────────────────────────────┘
```

**Conflict Resolution:** Higher levels override lower levels.

## Constitution Structure (Key Articles)

### Article I — Project Scope & Type

- Project type (Full-Stack, App-Only, Infra-Only)
- Active scopes (backend, frontend, cloud-platform, data, integration, ai, crm)

### Article X — Environments & Configuration

- Enabled environments (dev, uat, pre, prod)
- Configuration management strategy
- Feature flag provider

### Article XI — CI/CD Pipeline

- Platform (GitHub Actions, Azure DevOps)
- Pipeline stages (build, lint, test, security scan)
- Quality gate thresholds

### Article XII — Observability

- Monitoring strategy (Azure-Native, OpenTelemetry)
- Infrastructure monitoring components

### Article XVI — Security Policies

- Network security (VNet, Private Endpoints, WAF)
- Encryption at rest (Azure-managed vs Customer-managed keys)
- Compliance requirements (GDPR, HIPAA, SOC 2, PCI-DSS)

## Reading the Constitution

**AI Agent MUST:**

1. **Read constitution BEFORE generating code**

```bash
cat .aurora/memory/constitution.md
```

2. **Extract constraints** from relevant articles
3. **Validate output** against constitution rules
4. **Reject requests** that violate constitution

**Human Developer SHOULD:**

1. Read constitution before implementing features
2. Verify technology stack section for approved libraries
3. Check quality gate requirements before PR
4. Ensure architecture tests validate constitution rules

## Constitution Governance

**Stored in:** `.aurora/memory/constitution.md`
**Generated by:** `Init.ps1` / `init.sh` during project initialization
**Augmented by:** Scope-specific constitutions from `.aurora/scopes/*/memory/constitution.md`
**Decisions recorded in:** `.aurora/scopes.yaml` (wizard choices)
