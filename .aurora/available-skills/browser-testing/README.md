# Browser Testing Skills

Browser automation, visual testing, and UI validation skills using Playwright and browser automation tools.

## Skills in this Category

### playwright-comprehensive
**Comprehensive Playwright browser automation and testing toolkit**

Use for:
- **E2E Testing**: End-to-end test suites with Page Object Model, test reporters, and multi-browser support
- **Browser Automation**: General automation tasks like screenshots, form filling, web scraping
- **Local Webapp Testing**: Testing and debugging local development servers with auto-detection

Supports: Chromium, Firefox, WebKit, responsive design validation, API mocking, comprehensive test reporting.

### web-design-reviewer
**Visual inspection and design validation for websites**

Use for:
- Visual review of locally running websites or staging environments
- Identifying design issues, layout problems, responsive breakage
- Capturing screenshots across viewports for design QA
- Validating UI consistency and accessibility

Requires browser automation capabilities and access to running website.

## When to Use Browser Testing Skills

Choose skills from this category when you need to:
- Automate browser interactions for testing or data collection
- Validate UI/UX visually across browsers and viewports
- Test frontend applications end-to-end
- Debug and inspect running web applications
- Capture visual evidence (screenshots) for documentation or QA
