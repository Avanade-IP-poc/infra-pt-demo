## Descripción

`wsSMIServer` expone un conjunto de WebMethods SOAP sobre la base de datos `Alizes`. Es
consumido internamente por `SICAWeb` para obtener eventos de circuito, contar usuarios por
zona y gestionar tarjetas. Sin embargo, al no requerir ninguna autenticación, cualquier
cliente en la red puede invocar estos métodos y leer o modificar datos de usuarios, tarjetas,
familias y eventos de acceso.

## Partes interesadas e intereses

| Parte interesada       | Interés                                                         |
| ---------------------- | --------------------------------------------------------------- |
| SICAWeb                | Recuperar datos de Alizes sin acceso directo a BD               |
| Operación de Seguridad | La información de accesos y tarjetas debe estar protegida        |
| Auditoría              | Las modificaciones de datos deben ser trazables                 |
| Atacante potencial     | Sin auth → lectura/escritura libre de datos sensibles ⚠️       |

## Precondiciones

1. `wsSMIServer` está desplegado y accesible en la red.
2. La base de datos `Alizes` (`rfsql01`) es accesible.
3. (No hay más precondiciones — el servicio no requiere autenticación.)

## Postcondiciones (garantías de éxito)

1. El consumidor recibe los datos solicitados o confirmación de modificación.
2. La base de datos `Alizes` refleja los cambios realizados (si aplica).

## Disparadores

- Llamada SOAP desde `SICAWeb` (Acessos.ascx, Circuitos.ascx, Visitantes.ascx).
- Llamada SOAP desde cualquier cliente externo (no autenticado).

## Flujo principal de éxito — GetLastCircuitEvents (representativo)

| Paso | Actor      | Sistema                                                              |
| ---- | ---------- | -------------------------------------------------------------------- |
| 1    | SICAWeb    | Invoca `GetLastCircuitEvents(circuitId, startDate, endDate)`         |
| 2    |            | Sin verificación de autenticación ni autorización                    |
| 3    |            | Ejecuta SELECT sobre `Alizes.VueHisto` filtrado por circuito y fechas|
| 4    |            | Filtra eventos de tipo 249, 253, 255 (tipo desconocido)              |
| 5    |            | Devuelve array de objetos `CircuitEvent` al consumidor               |

## Extensiones (flujos alternativos)

### 2a. Cliente externo no autorizado (vector de ataque)

| Paso | Actor                     | Sistema                                               |
| ---- | ------------------------- | ----------------------------------------------------- |
| 2a.1 | Cliente externo en la red | Descubre el endpoint ASMX (`.asmx?wsdl`)              |
| 2a.2 |                           | El servicio no solicita credenciales                  |
| 2a.3 | Atacante                  | Invoca `GetUsers` → obtiene todos los usuarios        |
| 2a.4 | Atacante                  | Invoca `UpdateSmartCard` → desactiva tarjetas ⚠️     |
| 2a.5 | Atacante                  | Invoca `AddUserFamily` → modifica permisos de acceso  |

### 3a. Inyección SQL en parámetros SOAP

| Paso | Sistema                                                         |
| ---- | --------------------------------------------------------------- |
| 3a.1 | `LogicalCode` recibido contiene `' OR '1'='1`                   |
| 3a.2 | Consulta concatenada (`SMIMethods.asmx.vb:28`) devuelve todo    |
| 3a.3 | Datos de todos los usuarios filtrados incorrectamente           |

### *a. BD Alizes no disponible

| Paso | Sistema                                      |
| ---- | -------------------------------------------- |
| *a.1 | `InicializeConnSQL` falla                    |
| *a.2 | Sin manejo de excepción → SOAP fault genérico|

## Requisitos especiales

- **Seguridad CRÍTICA**: Todos los WebMethods deben requerir autenticación. ⚠️
  Sin autenticación, `AddUserFamily` y `UpdateSmartCard` permiten escalada de privilegios.
- **Auditoría**: Las modificaciones (AddUserFamily, UpdateSmartCard) no se registran en
  `tblLog`.
- **Inyección SQL**: `GetUsersByLogicalCode` concatena `LogicalCode` sin parametrizar
  ([SMIMethods.asmx.vb:28](../../../../../demo/from_old_src/wsSMIServer/SMIMethods.asmx.vb)).

## Frecuencia estimada

- Múltiples veces por minuto durante la operación normal (actualizaciones de UI con timer).
