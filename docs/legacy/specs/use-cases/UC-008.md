# Use Case: Análisis de Historial de Accesos por Zona y Período

## Metadata

## Descripción

Un supervisor autorizado analiza qué personas estuvieron en una o varias zonas durante un
período determinado, calculando horas de entrada y salida a partir de los eventos del log.
El acceso a esta funcionalidad está restringido a usuarios incluidos en la configuración
`AcessoAHistorico`.

## Partes interesadas e intereses

| Parte interesada       | Interés                                                         |
| ---------------------- | --------------------------------------------------------------- |
| Supervisor de seguridad| Reportes de ocupación/asistencia por zona y período             |
| RRHH / Auditoría       | Trazabilidad de presencia para control y cumplimiento           |
| Seguridad              | Solo usuarios autorizados acceden a datos históricos sensibles  |

## Precondiciones

1. El usuario está incluido en `AppSettings("AcessoAHistorico")`.
2. Existe acceso a la vista `vwREFERLog` y a `vwREFERCircuitos`.
3. El operador ha superado [UC-003](UC-003.md) (sesión válida).

## Postcondiciones (garantías de éxito)

1. Se muestra una tabla con las personas presentes en la zona y sus horas de entrada/salida.
2. El reporte cubre el período solicitado, ampliado para capturar pares entrada/salida.

## Disparadores

- Carga del control `LogHistorico.ascx`.
- Pulsación del botón "Consultar".

## Flujo principal de éxito

| Paso | Actor      | Sistema                                                                   |
| ---- | ---------- | ------------------------------------------------------------------------- |
| 1    | Supervisor | Accede al módulo de historial                                            |
| 2    |            | Verifica que el usuario está en `AcessoAHistorico` (RULE-008)            |
| 3    |            | Carga circuitos disponibles desde `vwREFERCircuitos`                      |
| 4    |            | Sugiere período por defecto (ayer → hoy)                                  |
| 5    | Supervisor | Selecciona circuitos y ajusta el período (inicio/fin)                     |
| 6    | Supervisor | Pulsa "Consultar"                                                        |
| 7    |            | Consulta `vwREFERLog` por circuitos y rango de fechas                     |
| 8    |            | Calcula entrada/salida emparejando eventos (códigos 130/133)             |
| 9    |            | Muestra tabla: Tarjeta · Nombre · Circuito · Entrada · Salida            |

### Detalle de pasos

**Paso 2 — Control de acceso (RULE-008)**

- `AppSettings("AcessoAHistorico")` contiene la lista de usuarios autorizados.
- Se compara contra `Session("Utilizador")`.

**Paso 7 — Consulta de log (⚠️ riesgo de seguridad)**

- ⚠️ **Inyección SQL CRÍTICA**: `LogHistorico.ascx.vb:24-30` concatena las fechas de
  `txtPeriodoParaAnaliseInicio.Text` y `txtPeriodoParaAnaliseFim.Text` sin parametrizar.

**Paso 8 — Cálculo entrada/salida**

- Amplía el rango ±7 días para capturar pares de entrada/salida que crucen los límites.
- Interpreta `descriptioncircuit` ("ENTRADA"/"SALIDA") o `parametre` (1=dentro, 2=fuera).

## Extensiones (flujos alternativos)

### 2a. Usuario no autorizado

| Paso | Actor      | Sistema                                                |
| ---- | ---------- | ------------------------------------------------------ |
| 2a.1 |            | `Session("Utilizador")` no está en `AcessoAHistorico`  |
| 2a.2 |            | Deniega el acceso al módulo                            |
| 2a.3 | Supervisor | No puede ejecutar consultas                            |

### 6a. Tipo de consulta alternativo

| Paso | Sistema                                                          |
| ---- | ---------------------------------------------------------------- |
| 6a.1 | El supervisor cambia `rblTipoConsulta`                           |
| 6a.2 | El sistema muestra/oculta paneles según el tipo de análisis      |

### 7a. Fecha con formato inválido / inyección

| Paso | Sistema                                                                |
| ---- | ---------------------------------------------------------------------- |
| 7a.1 | El texto de fecha no es convertible o contiene un payload              |
| 7a.2 | La consulta concatenada puede fallar o ser manipulada (⚠️ SQLi)        |

## Requisitos especiales

- **Seguridad CRÍTICA**: Las fechas deben parametrizarse. Concatenar entrada de usuario en
  `vwREFERLog` permite inyección SQL.
- **Autorización**: Restricción por lista de usuarios en `AcessoAHistorico`.
- **Rendimiento**: La ampliación de ±7 días puede aumentar el volumen consultado.

## Frecuencia estimada

- Bajo demanda (investigaciones, reportes periódicos).
