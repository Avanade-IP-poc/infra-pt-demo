# Use Case: Navegación y Control de Acceso por Menú

## Metadata

| Propiedad        | Valor                                                       |
| ---------------- | ----------------------------------------------------------- |
| UC ID            | UC-013                                                      |
| Reglas           | RULE-008, RULE-009 (autorización de terminal y permisos)    |
| Actor primario   | Operador de seguridad (usuario humano en terminal)          |
| Alcance          | Sistema — SICAWeb (`Menu.ascx`, `MonSeg.aspx`)              |
| Nivel            | User Goal                                                   |
| Estado           | Draft — extraído de legacy                                  |
| Fuente           | `demo/from_old_src/SICAWeb/SICAWeb/MonSeg.aspx.vb`          |

## Descripción

La página contenedora principal valida que el terminal sea válido y presenta las pestañas
funcionales (Log de Puertas, Resumen de Zonas, Histórico, Configuración de Accesos, Alarmas).
El menú resalta la sección activa y oculta las opciones según los permisos del usuario.

## Partes interesadas e intereses

| Parte interesada       | Interés                                                         |
| ---------------------- | --------------------------------------------------------------- |
| Operador de seguridad  | Navegar solo a las secciones permitidas                         |
| Seguridad              | Que cada usuario vea únicamente las funciones autorizadas       |

## Precondiciones

1. El operador ha superado [UC-003](UC-003.md) (`Session("NomeTerminal")` y `Session("Utilizador")`).

## Postcondiciones (garantías de éxito)

1. Se valida que el terminal existe en `tblTerminais`.
2. Se muestran solo las pestañas permitidas para el usuario.
3. La pestaña activa queda resaltada.

## Disparadores

- Carga de la página `MonSeg.aspx`.
- Selección de una pestaña del menú.

## Flujo principal de éxito

| Paso | Actor    | Sistema                                                                  |
| ---- | -------- | ------------------------------------------------------------------------ |
| 1    | Operador | Accede a la página principal de monitorización                          |
| 2    |          | Valida el terminal: `select count(*) from tblTerminais where nome=...`   |
| 3    |          | Evalúa permisos del usuario (`AcessoAHistorico`, `AcessoDeConfiguracao`) |
| 4    |          | Oculta las pestañas no permitidas en `Menu.ascx`                         |
| 5    |          | Renderiza la pestaña por defecto resaltada                               |
| 6    | Operador | Selecciona otra pestaña                                                  |
| 7    |          | `SelectTab(index)` cambia el contenido y resalta la nueva pestaña        |

### Detalle de pasos

**Paso 2 — Validación de terminal (⚠️ riesgo de seguridad)**

- ⚠️ **Inyección SQL MEDIA**: `MonSeg.aspx.vb:45,58` concatena `Session("NomeTerminal")` sin
  parametrizar sobre `tblTerminais` (manipulación posible vía tampering de sesión).

**Paso 3/4 — Permisos (RULE-008, RULE-009)**

- `Menu.ascx.vb` compara `Session("Utilizador")` contra `AcessoAHistorico` y
  `AcessoDeConfiguracao` para mostrar/ocultar las opciones de menú.

## Extensiones (flujos alternativos)

### 2a. Terminal no registrado

| Paso | Sistema                                                  |
| ---- | -------------------------------------------------------- |
| 2a.1 | El conteo sobre `tblTerminais` devuelve 0                |
| 2a.2 | El terminal no se considera válido                       |

### 3a. Usuario sin permisos especiales

| Paso | Sistema                                                          |
| ---- | ---------------------------------------------------------------- |
| 3a.1 | El usuario no está en `AcessoAHistorico` ni `AcessoDeConfiguracao` |
| 3a.2 | Solo se muestran las pestañas de monitorización básica            |

## Requisitos especiales

- **Seguridad MEDIA**: Parametrizar la validación de terminal; `Session("NomeTerminal")` nunca
  se valida tras el login.
- **Autorización**: Visibilidad de pestañas basada en listas de usuarios en `AppSettings`.

## Frecuencia estimada

- En cada carga de la aplicación (punto de entrada principal).

## Cuestiones abiertas

- ¿La gestión de permisos por listas en `AppSettings` debe migrar a roles (RBAC)?
- ¿Qué ocurre exactamente cuando el terminal no es válido (redirección, error)?
