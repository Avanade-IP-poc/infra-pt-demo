## Descripción

El operador selecciona un circuito (puerta/zona) y monitoriza en tiempo real los últimos
eventos de paso de tarjeta, viendo además los datos del último usuario que cruzó (foto,
nombre, empresa, zona, marca de tiempo). La vista se refresca automáticamente mediante un
temporizador configurable.

## Partes interesadas e intereses

| Parte interesada       | Interés                                                         |
| ---------------------- | --------------------------------------------------------------- |
| Operador de seguridad  | Vigilancia en tiempo real de accesos por puerta                 |
| Gestión de seguridad   | Detección temprana de accesos sospechosos                       |

## Precondiciones

1. El operador ha superado [UC-003](UC-003.md) (sesión con `Session("NomeTerminal")`).
2. Existen circuitos asociados al terminal en `vwCircuitos` / `tblCircuitos`.
3. El servicio SOAP `wsSMIServer` está disponible.

## Postcondiciones (garantías de éxito)

1. Se muestran los últimos eventos del circuito seleccionado (máx. 20, ventana 72 h).
2. Se muestran los datos del último usuario que pasó.
3. La vista se mantiene actualizada según el intervalo de refresco configurado.

## Disparadores

- Carga del control `Circuitos.ascx`.
- Selección de un circuito en el desplegable.
- Tick del temporizador de refresco.
- Pulsación del botón de refresco manual.

## Flujo principal de éxito

| Paso | Actor    | Sistema                                                                       |
| ---- | -------- | ----------------------------------------------------------------------------- |
| 1    | Operador | Accede al monitor de circuitos                                                |
| 2    |          | Carga los circuitos del terminal vía `SelectVwCircuitosByNomeTerminal` (SP)   |
| 3    | Operador | Selecciona un circuito en el desplegable                                       |
| 4    |          | Guarda `Session("CircuitoSeleccionado")` y mapea a circuitos físicos          |
| 5    |          | Invoca SOAP `GetLastCircuitEvents(circuitIDs, 72, 20)`                        |
| 6    |          | Muestra la lista de eventos y el detalle del último usuario                   |
| 7    |          | Cada N segundos (`DisplayLogRefreshInterval`), refresca automáticamente        |

### Detalle de pasos

**Paso 2 — Carga de circuitos**

- SP parametrizado `SelectVwCircuitosByNomeTerminal` con `@NomeTerminal` (seguro).
- Guarda el resultado en `Session("Circuitos")` y `Session("CircuitosFisicos")`.

**Paso 5 — Eventos vía SOAP**

- `wsSMIServer.GetLastCircuitEvents` con ventana de 72 h y máximo 20 eventos.
- El detalle del último usuario (`DetalheUtilizador.ascx`) muestra la foto obtenida de AD.

## Extensiones (flujos alternativos)

### 2a. Circuitos no cargados en sesión (primer acceso)

| Paso | Sistema                                                          |
| ---- | ---------------------------------------------------------------- |
| 2a.1 | `Session("Circuitos")` está vacío                                |
| 2a.2 | Ejecuta consulta sobre `tblCircuitos` filtrando por `IDCircuitoGrupo` |
| 2a.3 | Cachea el resultado en sesión                                    |

### 5a. Servicio SOAP no disponible

| Paso | Sistema                                                  |
| ---- | -------------------------------------------------------- |
| 5a.1 | `GetLastCircuitEvents` lanza excepción                   |
| 5a.2 | Sin manejo explícito → la lista de eventos queda vacía   |

### 6a. Foto del último usuario no disponible

| Paso | Sistema                                                                          |
| ---- | -------------------------------------------------------------------------------- |
| 6a.1 | `DetalheUtilizador` consulta `tblAD_AD_SQL` por `employeeID` (⚠️ SQLi en L32)    |
| 6a.2 | Si no hay foto (`wWWHomePage`), muestra imagen por defecto                        |

## Requisitos especiales

- **Rendimiento**: Refresco periódico; ventana acotada a 72 h y 20 eventos para limitar carga.
- **Seguridad**: `DetalheUtilizador.ascx.vb:32` concatena `NumEmpregado` sin parametrizar
  (riesgo medio si el log contiene payloads).

## Frecuencia estimada

- Monitorización continua durante el turno del operador.
