## Descripción

El operador visualiza un panel con el conteo de personas presentes en cada zona en tiempo
real. Puede expandir una zona concreta para ver el listado detallado de las personas
presentes. El panel se refresca automáticamente.

## Partes interesadas e intereses

| Parte interesada       | Interés                                                         |
| ---------------------- | --------------------------------------------------------------- |
| Operador de seguridad  | Visión global de ocupación por zona en tiempo real              |
| Gestión de seguridad   | Control de aforo y evacuación                                   |

## Precondiciones

1. El operador ha superado [UC-003](UC-003.md) (sesión válida).
2. El servicio SOAP `wsSMIServer` está disponible.

## Postcondiciones (garantías de éxito)

1. Se muestra el conteo de personas por zona.
2. Al seleccionar una zona, se muestra el detalle de las personas presentes.
3. El panel se mantiene actualizado según el intervalo configurado.

## Disparadores

- Carga del control `ResumoZonas.ascx`.
- Selección de una zona en el grid.
- Tick del temporizador de refresco.

## Flujo principal de éxito

| Paso | Actor    | Sistema                                                              |
| ---- | -------- | ------------------------------------------------------------------- |
| 1    | Operador | Accede al dashboard de zonas                                        |
| 2    |          | Invoca SOAP `CountUsersByZone()`                                    |
| 3    |          | Muestra el grid con el conteo por zona                              |
| 4    | Operador | Selecciona una zona                                                 |
| 5    |          | Invoca SOAP `GetUsersByZone(idZona)`                               |
| 6    |          | Muestra el detalle de personas presentes (`DetalheZona.ascx`)       |
| 7    |          | Cada N segundos (`ZonasRefreshInterval`), refresca el conteo         |

### Detalle de pasos

**Paso 2 y 5 — Datos vía SOAP**

- `CountUsersByZone()` devuelve el agregado por `IDZona`.
- `GetUsersByZone(idZona)` devuelve la lista de presentes en la zona seleccionada.

## Extensiones (flujos alternativos)

### 4a. Cierre del panel de detalle

| Paso | Actor    | Sistema                                          |
| ---- | -------- | ------------------------------------------------ |
| 4a.1 | Operador | Pulsa el botón de cierre del detalle             |
| 4a.2 |          | Oculta el panel de detalle de zona               |

### 5a. Servicio SOAP no disponible

| Paso | Sistema                                                  |
| ---- | -------------------------------------------------------- |
| 5a.1 | `GetUsersByZone` / `CountUsersByZone` lanza excepción    |
| 5a.2 | Sin manejo explícito → el panel queda sin datos          |

## Requisitos especiales

- **Tiempo real**: El conteo debe reflejar el estado actual con baja latencia.
- **Seguridad**: Componente sin SQL directo (todo vía SOAP) — sin riesgo de inyección.

## Frecuencia estimada

- Visualización continua durante el turno.
