## Descripción

El responsable de recepción consulta el listado de tarjetas de visitante asignadas/disponibles
para el terminal actual. Desde el listado puede seleccionar una tarjeta para abrir su detalle
y editarla ([UC-011](UC-011.md)). El listado se refresca automáticamente.

## Partes interesadas e intereses

| Parte interesada       | Interés                                                         |
| ---------------------- | --------------------------------------------------------------- |
| Responsable recepción  | Visión de las tarjetas de visitante gestionables en el terminal |
| Seguridad              | Control de qué tarjetas están disponibles por terminal          |

## Precondiciones

1. El operador ha superado [UC-003](UC-003.md) (`Session("NomeTerminal")` establecido).
2. Existen familias y tarjetas asociadas al terminal.

## Postcondiciones (garantías de éxito)

1. Se muestra el listado de tarjetas de visitante del terminal.
2. El listado refleja el estado actual y se refresca periódicamente.

## Disparadores

- Carga del control `Visitantes.ascx`.
- Tick del temporizador `TimerCartoesFora`.
- Selección de una fila del grid.

## Flujo principal de éxito

| Paso | Actor      | Sistema                                                                |
| ---- | ---------- | ---------------------------------------------------------------------- |
| 1    | Recepción  | Accede al módulo de visitantes                                        |
| 2    |            | Carga las familias del terminal en `Session("FamiliasTerminal")`      |
| 3    |            | Carga las tarjetas del terminal en `Session("CartoesTerminal")`       |
| 4    |            | Ejecuta `sp_tblVisitantes_SelectAtribuidosByNomeTerminal` (SP)        |
| 5    |            | Muestra el grid de tarjetas y configura el temporizador de refresco    |
| 6    | Recepción  | Selecciona una tarjeta                                                |
| 7    |            | Abre el detalle de visitante ([UC-011](UC-011.md))                    |

### Detalle de pasos

**Paso 2/3 — Carga de familias y tarjetas (⚠️ riesgo de seguridad)**

- ⚠️ **Inyección SQL ALTA**: `Visitantes.ascx.vb:27,31,110` concatena `Host` y `NomeTerminal`
  sin parametrizar en consultas a `vwCartoes` y `vwFamilias`.

**Paso 4 — Tarjetas atribuidas**

- SP parametrizado (seguro) que lista las tarjetas asignadas al terminal.

## Extensiones (flujos alternativos)

### 5a. Refresco automático

| Paso | Sistema                                                          |
| ---- | ---------------------------------------------------------------- |
| 5a.1 | Cada N segundos (`CartoesForaRefreshInterval`) refresca el grid  |
| 5a.2 | Muestra la marca de tiempo si `ShowRefreshTime` está activo       |

### 2a. Terminal identificado por IP o por nombre

| Paso | Sistema                                                          |
| ---- | ---------------------------------------------------------------- |
| 2a.1 | Según el tipo de host, filtra por `IPTerminal` o por `NomeTerminal` |

## Requisitos especiales

- **Seguridad ALTA**: Parametrizar `vwCartoes`/`vwFamilias`; la concatenación de `Host` y
  `NomeTerminal` permite inyección SQL aunque provengan de sesión.
- **Rendimiento**: Refresco periódico del listado de tarjetas.

## Frecuencia estimada

- Uso frecuente en recepción durante el horario de visitas.
